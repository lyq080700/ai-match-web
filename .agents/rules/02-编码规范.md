# 编码规范

## TypeScript 规范

- 所有代码必须使用 TypeScript，禁止使用 JavaScript
- 所有 `.ts` 和 `.tsx` 文件必须包含完整的类型定义
- 禁止使用 `any` 类型（除非有明确理由并添加注释说明）
- 接口、类型定义必须使用 PascalCase 命名
- 组件 Props 必须定义清晰的 TypeScript 接口

## 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 文件夹/路由 | kebab-case | `user-profile`、`login` |
| 变量/函数 | camelCase | `userList`、`onSubmit` |
| 常量 | UPPER_CASE | `API_TIMEOUT` |
| 枚举及属性 | PascalCase / UPPER_CASE | `ApiMethod.GET` |
| 接口/类 | PascalCase | `UserInfo`、`BaseResponse` |
| React 组件 | PascalCase | `UserProfile` |
| 自定义 Hooks | use 开头 | `useUserStore` |

## 业务函数命名

- **事件处理函数**：`onXxx`（如 `onBannerClick`、`onFormSubmit`）
- **内部处理函数**：`handleXxx`（如 `handleDelete`、`handleSubmit`）
