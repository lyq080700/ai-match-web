# 项目结构（NON-NEGOTIABLE）

## 目录结构

项目采用 Next.js 16 App Router 模式：

```
ai-match-web/
├── app/                    # Next.js App Router
│   ├── (auth)/             # 认证相关路由组
│   │   ├── login/
│   │   └── register/
│   ├── (main)/             # 主业务路由组
│   │   └── ...
│   ├── globals.scss        # 全局样式
│   ├── layout.tsx          # 根布局
│   └── page.tsx            # 首页
├── components/            # 可复用 UI 组件（与 app 平级）
│   └── xxx/
│       ├── index.tsx
│       └── index.module.scss
├── hooks/                  # 自定义 Hook
├── services/               # 请求服务
├── interfaces/             # 接口、模型、DTO 类型声明
├── stores/                 # Zustand store，按业务拆分
├── utils/                  # 通用工具函数
└── common/                 # 业务常量、枚举、静态资源
```

## 结构约束

| 类型     | 目录          | 规范                                                          |
| -------- | ------------- | ------------------------------------------------------------- |
| 页面     | `app/`        | 使用 App Router 目录结构，layout.tsx / page.tsx / loading.tsx |
| 组件     | `components/` | 每个组件独立目录，导出到 `index.ts`                           |
| 常量     | `common/`     | 按模块划分，所有模块导出到 `index.ts`                         |
| 状态     | `stores/`     | 按业务模块划分，暴露 `useXxxStore` Hook                       |
| 工具     | `utils/`      | 按功能模块划分，所有模块导出到 `index.ts`                     |
| 接口类型 | `interfaces/` | 每个模块含 `model.ts` 和 `api.ts`                             |
| Hooks    | `hooks/`      | 自定义 Hook，按功能命名                                       |

## 路由目录内的组件放置规则

页面专用组件放在页面目录下的 `components/` 目录：

```
app/(main)/xxx/
├── components/           # 仅供当前页面使用的子组件
│   └─ xxx/
│       ├─ index.tsx
│       └─ index.module.scss
├── page.tsx
├── layout.tsx
└─ loading.tsx
```

**判断标准**：

- 通用组件：多处复用、业务无关 → `components/`
- 页面级组件：仅单页使用 → 路由目录下的 `components/`

**约束**：`components/` 只存放通用组件，禁止将仅单页使用的组件放入。
