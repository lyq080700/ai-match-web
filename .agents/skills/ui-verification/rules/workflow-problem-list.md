## UI 问题清单文档结构

### 文档路径

```
docs/样式还原/<名称>-UI问题清单.md
```

### 文档结构建议

```markdown
# <名称> - UI 问题清单

> **参照文档**：`docs/样式还原/<名称>-UI分析清单.md`  
> **设计稿**：`docs/设计稿/<路径>/<文件名>.pen` 或 Figma 链接  
> **创建/更新日期**：YYYY-MM-DD

## 一、按区域列问题（区域按从上到下、从左到右顺序排列）

### 区域 A
| 编号 | 问题描述 | 类型 | 当前实现 | 设计稿要求 | 优先级 |
|------|----------|------|----------|------------|--------|
| A1   | ...      | ...  | ...      | ...        | P0/P1/P2 |

## 二、问题统计与修复顺序

| 优先级 | 数量 | 说明 |
|--------|------|------|
| P0（布局/层级/内容） | n | 先修，不得遗漏 |
| P1 | n | 次之 |
| P2 | n | 可后续 |

建议修复顺序：P0 → P1 → P2；同优先级内按**从上到下、从左到右、从外到里**。

## 三、验收结论（随修复更新）

- [ ] **P0（布局、层级、文字、图片）** 全部修复，且验收时已按从上到下、从左到右、从外到里逐项检查无遗漏
- [ ] P1 全部修复
- [ ] P2 已记录/部分修复
- [ ] 已使用 Browser 进行实际页面与设计稿比对
- [ ] 还原度自评：___%
```

### 问题描述要求

- **问题描述**：简短、可操作（如「欢迎语区域整体水平居中」）
- **当前实现**：现状（如 `align-items: center`）
- **设计稿/分析清单要求**：明确要求（如「左对齐 flex-start」）
- **优先级**：便于排期与迭代
