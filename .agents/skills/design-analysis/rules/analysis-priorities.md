# 文字、图片、布局、层级四类重中之重

## 核心原则

**文字、图片、布局、层级是页面元素的四类重中之重，均不允许出错或遗漏。**

不按顺序分析，或漏记、错记文字/图片/布局/层级，是分析遗漏与还原错误的主要来源。**严格按「从上到下、从左到右、从外到里」并逐条准确记录这四者，才能保证分析完整且实现不出错。**

## 一、文字

### 记录要求

**所有可见文案都必须记录**，包括：
- 欢迎语、标题、副标题、说明文字
- 按钮/链接文案
- 占位文案
- Tab 文字
- 标签文字
- 任何可见的文字内容

### 记录项

每处文字需记录：
- **内容**：完整或典型示例
- **字体**：fontFamily
- **字号**：fontSize
- **字重**：fontWeight
- **颜色**：color（含透明度）
- **行高**：lineHeight
- **字间距**：letterSpacing（若有）

### 易漏点

- 页面最上方的欢迎语/大标题（最容易在「只记了下面小字」时被漏掉）
- 副标题
- 按钮/链接文案
- 占位提示文字
- Tab 或标签文字

**检查方法**：按「从上到下」扫到时立即记下，不要等到后面再补。

## 二、图片

### 记录要求

**所有图片、占位图、图标区域都必须记录**，包括：
- 图片
- 占位图
- 图标区域
- 任何可见的图像内容

### 记录项

每处图片需记录：
- **尺寸**：宽×高（width × height）
- **位置**：相对父级或相邻元素的 x, y
- **比例**：宽高比
- **圆角**：cornerRadius（如有）
- **与相邻元素的间距**：margin 或 gap
- **多图拼接时的布局关系**：左右/上下/网格，以及每张的占比

### 易漏点

- 小图标
- 占位图
- 多图拼接时的布局关系
- 图片的圆角和比例

**检查方法**：逐区域扫描，确保每张图片都有记录。

## 三、布局

### 记录要求

**区域与元素的位置、尺寸、间距、对齐方式、排列方向、多图拼接方式等必须准确记录，与设计稿一致，不可错位或比例失调。**

### 记录项

- **位置**：x, y（相对父级）
- **尺寸**：width, height
- **间距**：padding, gap, margin（含方向）
- **对齐方式**：align-items, justifyContent
- **排列方向**：横向/纵向/网格
- **多图拼接方式**：左右/上下/网格布局

### 易漏点

- 间距（padding/gap/margin）的方向性
- 对齐方式（左对齐/居中/右对齐）
- 排列方向（横向/纵向/网格）
- 多图拼接时的布局方式

**检查方法**：对照设计稿，确保布局参数与设计稿一致。

## 四、层级

### 记录要求

**从外到里的嵌套关系（谁包谁、父子/兄弟）、区块的先后顺序与叠放关系必须写清，保证实现时结构正确、不丢层或顺序颠倒。**

### 记录项

- **嵌套关系**：谁包谁、父子/兄弟关系
- **先后顺序**：区块的显示顺序
- **叠放关系**：z-index 或层叠顺序

### 易漏点

- 外层容器被忽略
- 嵌套层级丢失
- 顺序颠倒
- 叠放关系不明确

**检查方法**：按「从外到里」逐层分析，确保每层都有记录。

## 检查清单

分析时必过一遍：

- [ ] **所有可见文字**：欢迎语、大标题、副标题、说明、按钮/链接/占位/Tab 文案等，按从上到下逐条核对
- [ ] **所有图片/占位图**：尺寸、位置、比例、多图拼接布局，无遗漏
- [ ] **布局**：位置、尺寸、间距、对齐、排列方向、多图拼接方式与设计稿一致，无错位或比例失调
- [ ] **层级**：从外到里的嵌套与先后顺序正确，无丢层或顺序颠倒
