# 第四步：输出文档要求

## 概述

第四步是将分析结果输出为「UI 分析清单」文档，供开发与验收参照。本文档说明文档路径、结构和质量要求。

## 4.1 文档路径

```
docs/样式还原/<页面或模块名称>-UI分析清单.md
```

**示例**：
- `docs/样式还原/用户中心-UI分析清单.md`
- `docs/样式还原/商品列表页-UI分析清单.md`

## 4.2 文档结构（必须包含）

文档必须包含以下章节，详见 `output-analysis-checklist.md` 获取完整模板：

1. **文档说明**：设计稿路径/链接、分析日期
2. **一、设计稿整体分析**：页面状态、整体尺寸、区域划分、区域间间距
3. **二、区域详细分析**：按从上到下、从左到右、从外到里顺序，每个区域包含所有文字、图片、布局、层级
4. **三、样式规范汇总**：颜色、字体、圆角、间距、阴影、效果
5. **四、交互状态分析**：默认 / hover / active / 禁用等（如有）
6. **五、占位元素说明**：图标占位尺寸与位置、图片占位尺寸与比例
7. **六、常见遗漏检查点**：本次分析已核对项，可列 checklist
8. **七、实现建议**：组件划分、主流网页设计常识、常见错误模式
9. **八、验证检查清单**：后续验收时用的布局/样式/元素/交互检查项
10. **九、备注**：坐标系统、响应式、占位策略等

## 4.3 文档质量要求

| 要求 | 说明 |
|------|------|
| **分析顺序** | 区域与章节按「从上到下、从左到右、从外到里」书写，与前端布局一致，避免漏掉顶部或某一侧 |
| **文字、图片、布局、层级** | 四者为页面元素重中之重，均不允许出错：所有文字与图片无遗漏；布局（位置/尺寸/间距/对齐）与设计稿一致；层级（嵌套与先后）正确无丢层或颠倒 |
| 完整性 | 覆盖设计稿中所有关键区域与关键元素，无「略」式省略 |
| 准确性 | 尺寸、颜色、字号等从设计稿中读取并写明，不猜测 |
| 可执行 | 开发仅凭此文档即可实现布局与样式，无需反复看图 |
| 可维护 | 设计稿变更时，同步更新本文档并标注更新日期 |

## 质量检查清单

输出文档前，检查以下项：

- [ ] 文档路径正确，符合命名规范
- [ ] 文档结构完整，包含所有必需章节
- [ ] 分析顺序正确（从上到下、从左到右、从外到里）
- [ ] 所有文字、图片、布局、层级都已记录，无遗漏
- [ ] 样式规范汇总完整，包含颜色、字体、圆角、间距、阴影
- [ ] 实现建议包含组件划分、设计常识、常见错误模式
- [ ] 文档可执行，开发人员无需反复看图即可实现
- [ ] 文档可维护，设计稿变更时可同步更新
