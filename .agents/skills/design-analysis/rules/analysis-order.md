# 分析顺序（必守）

## 重要性

**必须按以下顺序阅读与记录设计稿，否则容易产生疏漏。**

不按顺序分析是分析遗漏与还原错误的主要来源之一。严格按顺序分析，才能保证分析完整且实现不出错。

## 分析顺序

### 1. 从上到下

先按垂直方向，从页面最上方扫到最下方，确定所有横向「带」与区块的先后顺序（与前端布局的从上至下一致）。

**为什么重要**：
- 与前端布局顺序一致，避免实现时顺序颠倒
- 确保顶部或首屏的大字号文案（如欢迎语）不被遗漏
- 避免只关注页面下方内容而漏掉顶部区域

### 2. 从左到右

在同一行或同一层级内，再按从左到右依次记录，不跳过、不凭印象合并。

**为什么重要**：
- 避免只记录左侧内容而漏掉右侧或对称区域
- 确保同一层级内的元素顺序正确
- 避免凭印象合并相似元素，导致细节丢失

### 3. 从外到里

在每个区块内部，按「外层容器 → 内层容器 → 叶子元素」逐层进入，先轮廓后细节。

**为什么重要**：
- 确保层级嵌套关系正确，不丢层或顺序颠倒
- 先理解整体结构，再深入细节
- 避免只关注细节而忽略整体布局

## 扫描顺序示例

**设计稿扫描**：
1. 先按 y 坐标从 0 起向下（或从大到小）确定所有横向区域顺序
2. 再在同一行内按 x 坐标从左到右读取
3. 使用 Pencil MCP 或 Figma MCP 时也按此顺序逐层获取布局与节点信息

**记录顺序**：
- 区域列表：按从上到下（y）排序
- 同层元素：从左到右（x）排序
- 嵌套结构：从外到里逐层进入
